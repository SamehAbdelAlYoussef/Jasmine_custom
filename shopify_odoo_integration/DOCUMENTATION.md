## ١. نظرة عامة على النظام

### ما هو الموديول؟

الموديول عبارة عن **جسر تكامل ثنائي الاتجاه** بين متجر Shopify الإلكتروني ونظام Odoo ERP. يقوم بمزامنة:

| من Shopify → إلى Odoo | من Odoo → إلى Shopify |
|---|---|
| 📦 الطلبات (Orders) | 📊 المخزون (Inventory) |
| 👤 العملاء (Customers) | ⚙️ إعدادات الشحن |
| 💰 المدفوعات (Payments) | |
| 🚚 الشحن (Shipping) | |

---

## ٢. المميزات الرئيسية

### 📦 أ. مزامنة الطلبات (Shopify → Odoo)

```
متجر Shopify ← كل ٥ دقايق أو Webhook فوري → Odoo Sale Order
```

- **تلقائي بالكامل**: كل طلب جديد في Shopify يتحول إلى أمر بيع (Sale Order) في Odoo
- **Webhook فوري**: الطلبات الجديدة والمعدلة والملغية تنعكس فوراً في Odoo
- **منع التكرار**: النظام يضمن عدم تكرار أي طلب (حماية Idempotency)
- **العملاء تلقائياً**: بيانات العميل (الاسم، العنوان، الهاتف) تنشئ أو تحدث تلقائياً
- **المنتجات تلقائياً**: المنتجات غير الموجودة في Odoo تُعلم بعلامة "منتج مفقود"
- **المدفوعات**: معاملات الدفع (Visa, COD, Paymob) تتزامن مع Odoo Payments

### 📊 ب. مزامنة المخزون (Odoo → Shopify)

```
تغيير مخزون Odoo ← طابور ذكي ← كل دقيقتين ← Shopify Inventory
```

- **لحظي في الخلفية**: أي تغيير في المخزون (بيع، شراء، جرد) يسجل تلقائياً
- **تحديث دفعي**: ٢٠٠ منتج كل دقيقتين للضمان عدم الضغط على API
- **ربط ذكي بالـ SKU**: المنتجات تتطابق تلقائياً بين النظامين بالباركود
- **المنتج مش موجود في Shopify؟** ← لا مشكلة، بيتخطاه تلقائياً
- **مراقبة كاملة**: لوحة تحكم لمتابعة حالة المزامنة (نجاح/فشل/قيد الانتظار)

### 🚚 ج. معالجة ذكية للشحن

- **الشحن المجاني**: النظام يكتشف الخصومات على الشحن (مثلاً: شحن ٨٠ ج + خصم شحن مجاني ٨٠- = صافي ٠) ويطبقها بدقة
- **منتج خدمي**: الشحن يضاف كمنتج خدمة (Service Product) لا يؤثر على المخزون
- **إعدادات الشحن**: إمكانية تحديد المنتجات التي تتطلب شحن فعلي

### 🔄 د. مزامنة شاملة (Full Sync)

- زر واحد يمسح **كل منتجات Shopify** ويطابقها مع Odoo
- ينشئ روابط الربط (Bindings) لكل المنتجات دفعة واحدة
- يضع كل المنتجات في طابور المزامنة للمعالجة في الخلفية

## ٣. كيف يعمل النظام؟

### الرسم البياني للتدفق

```
┌─────────────────────────────────────────────────────────────────────┐
│                        SHOPIFY STORE                                │
│  • العملاء يطلبون من المتجر                                         │
│  • المخزون يُعرض للعملاء                                            │
│  • المدفوعات تتم عبر بوابات الدفع                                    │
└────────────┬────────────────────────────────────┬───────────────────┘
             │                                    │
             │  أوامر، عملاء، مدفوعات             │  المخزون
             │  (Shopify → Odoo)                  │  (Odoo → Shopify)
             │                                    │
             ▼                                    │
┌─────────────────────────┐            ┌──────────┴──────────────────┐
│   Webhook فوري          │            │   طابور المزامنة            │
│   أو Cron كل ٥ دقائق    │            │   Stock Sync Queue          │
└───────────┬─────────────┘            └──────────┬──────────────────┘
            │                                     │
            ▼                                     ▼
┌─────────────────────────────────────────────────────────────────────┐
│                          ODOO ERP                                   │
│  • Sale Orders (أوامر البيع)                                        │
│  • Customers (العملاء)                                              │
│  • Payments (المدفوعات)                                             │
│  • Inventory / Stock (المخزون)                                      │
│  • Invoicing (الفواتير)                                             │
└─────────────────────────────────────────────────────────────────────┘
```

### دورة حياة الطلب

1. **العميل يشتري** من متجر Shopify
2. **Webhook** يوصل الطلب لـ Odoo فوراً (أو Cron يجمعه)
3. **Odoo ينشئ** Sale Order + Customer + Payment
4. **المخزون يتحجز** في Odoo
5. **توصيل الطلب** ← المخزون الفعلي ينقص
6. **الطابور يسجل** التغيير
7. **Cron يرسل** المخزون الجديد لـ Shopify

### دورة حياة المخزون

1. **استلام بضاعة** في Odoo (Purchase Order)
2. **المخزون يزيد** ← stock.quant يتغير
3. **الطابور يسجل**: "المنتج X كميته الجديدة = Y"
4. **كل دقيقتين**: الـ Cron يبعت القيم الجديدة لـ Shopify
5. **Shopify**: يعرض المخزون المحدث للعملاء

## ٤. الإعدادات المطلوبة

### الإعداد مرة واحدة فقط:

| الإعداد | الوصف |
|---|---|
| 🔗 Shopify Shop URL | رابط المتجر (`mystore.myshopify.com`) |
| 🔑 Access Token | رمز API من Shopify Admin |
| 📍 Location ID | رقم موقع المخزون (يتم جلبه تلقائياً بزر "Fetch") |
| ✅ Enable Stock Sync | تفعيل مزامنة المخزون |
| ⏱ Stock Sync Cron | تفعيل الجدولة (كل دقيقتين) |

## ٥. لوحات المراقبة

| الشاشة | الوصف |
|---|---|
| **Shopify Sync** | متابعة حالة مزامنة الطلبات (قيد التشغيل، مكتمل، أخطاء) |
| **Product Bindings** | عرض كل المنتجات المربوطة بين Odoo و Shopify |
| **Stock Sync Queue** | متابعة طابور مزامنة المخزون (المنتظرة، قيد المعالجة، الأخطاء) |

## ٦. المزايا التقنية

- ✅ **بدون اعتماديات خارجية**: لا يحتاج مكتبات OCA أو خدمات خارجية
- ✅ **حماية من الأعطال**: Auto-locking يمنع تشغيل مزامنتين في نفس الوقت
- ✅ **احترام حدود API**: نظام ذكي لتجنب تجاوز حدود Shopify API
- ✅ **أمان كامل**: لا يشارك بيانات مع أطراف خارجية
- ✅ **سجلات تفصيلية**: Logging كامل لكل عملية لتتبع الأخطاء
- ✅ **متوافق مع Odoo 19**: مبني بأحدث معايير Odoo
